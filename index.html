What is CI/CD?

CI/CD stands for Continuous Integration and Continuous Delivery/Deployment. Continuous Integration means developers frequently merge their code changes into a shared repository and run automated checks to detect issues early. Continuous Delivery/Deployment then takes the validated code and automatically prepares (or releases) it to production. This approach reduces manual work, improves reliability, and speeds up updates in cloud systems.

My Pipeline

After I push my code to the main branch, GitHub Actions runs a workflow that checks the required files and uploads the website files. Then it deploys the site automatically to GitHub Pages, so the live website updates with every successful push.
